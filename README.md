# 极速枪战 Arena

在线 **1v1 第一人称 3D 枪战对决**，手机浏览器打开即可游玩。Three.js 渲染 + Socket.IO 实时联机，**关键战斗数据由服务端权威裁决**（防前端作弊）。

## 玩法

- **单人训练**：首页选难度（简单 / 普通 / 困难）→ 点「开始单人对战 AI」立即开打，和一个服务端 AI 机器人 1v1。
- **联机对战**：创建房间 → 把 4 位房间号发给好友 → 好友输入房间号加入。
- 满 2 人（或单人 + AI）自动 3 秒倒计时开战，进入同一张对称竞技场。
- 击败对手 +1 分，被击败后 3 秒复活（复活有 1 秒无敌保护）。
- 单局 **3 分钟**，比分高者获胜；**平局进入 30 秒加时赛**，先击败对方者获胜。
- 武器：未来感突击步枪，单发 20 伤害 / 生命 100 / 弹匣 30 发 / 换弹 1.5 秒 / 子弹轻微散布。

### AI 敌人（单人模式）

AI 作为**服务端机器人玩家**接入，和真人共用同一套权威战斗逻辑（命中判定、扣血、击杀、复活、计分、加时），行为完全对称。三档难度调节反应速度、瞄准精度、射速与走位积极性：

| 难度 | 反应 | 瞄准 | 走位 |
|---|---|---|---|
| 简单 | 慢 | 误差大 | 保守 |
| 普通 | 适中 | 适中 | 适中 |
| 困难 | 快 | 精准 | 激进绕后侧移 |


## 操作（移动端）

| 控件 | 位置 | 作用 |
|---|---|---|
| 虚拟摇杆 | 左下角 | 前后左右移动 |
| 滑动 | 屏幕右侧 | 转动视角 / 瞄准 |
| 射击 | 右下角大按钮 | 开火 |
| 换弹 | 射击上方 | 装填（期间不能射击）|
| 跳 | 右侧 | 跳跃越障 |

桌面端调试：`WASD` 移动，鼠标点击画面锁定指针后移动鼠标转向，左键射击，`R` 换弹，`空格` 跳。

## 运行方式

需要 Node.js ≥ 16。

```bash
cd arena-fps
npm install      # 安装 express + socket.io
npm start        # 启动服务端（默认端口 3000，可用 PORT 环境变量覆盖）
```

启动后控制台会打印访问地址。

### 本机测试
浏览器打开两个标签页访问 `http://localhost:3000`，一个创建房间、一个加入即可对战。

### 手机对战（同一局域网）
1. 确保电脑和两台手机连同一个 Wi-Fi。
2. 查看电脑局域网 IP：
   - macOS：`ipconfig getifaddr en0`
   - Windows：`ipconfig` 找 IPv4 地址
3. 两台手机浏览器访问 `http://<电脑IP>:3000`，一台创建、一台输房间号加入。

> 进入对战时会尝试请求全屏与横屏锁定，竖屏同样可玩（优先横屏体验更佳）。

## 联机与反作弊设计

- **移动/朝向**：客户端预测并上报，服务端转发给对手（低延迟、即时反馈）。
- **战斗裁决（服务端权威）**：射击只发送方向，服务端用共享地图做
  「射线 vs 玩家圆柱 + 掩体遮挡」判定，自行扣血、记分、判定击杀与复活；
  客户端无法直接修改血量/比分。
- **限制**：服务端校验弹药、换弹状态、射速上限（防连点）、房间最多 2 人、加时先击杀即胜。
- **异常处理**：断线进入 10 秒宽限可重连；超时或主动退出则对手收到「对手已离开房间」并结束对局；
  空房间定时回收，重复/超员加入被拒绝。

## 性能

- 移动端优先：本地 Three.js、简单材质、低面数模型、粒子数量受控、阴影贴图 1024、像素比上限 2。
- 帧率目标 30 FPS 以上，低性能设备也可运行。

## 测试

```bash
# 终端 1：启动测试服务端
PORT=3010 node server.js

# 终端 2：服务端联机流程测试
npm test                       # = node test/integration.js

# 终端 2：浏览器端烟测（需本机安装 Chrome）
node test/browser-smoke.js
```

> `test/` 依赖 `socket.io-client` 与 `puppeteer-core`（已在 devDependencies）。
> 烟测使用系统 Chrome 的软件 WebGL，可用 `CHROME=/path/to/chrome` 覆盖路径。
